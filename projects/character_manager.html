Gestionnaire de personnages

Informations générales :

  • Type de projet : Application web
  • Langages utilisés : Python, SQL, HTML, CSS, Javascript
  • Technologies utilisées : Flask, Sqlite3, Jinja2
  • Période de développement : 03/2024 - 04/2024
  • Contexte : Licence 2 Informatique - UE Web
  • Statut : Terminé

Description du projet :

     Cette application de gestion de personnages est conçue selon une architecture MVC (Model View Controller), avec l'utilisation du module Flask pour le serveur, du module Sqlite3 pour le modèle de données et de templates Jinja2 pour les vues. Les données sont stockées dans une base de données SQLite.

     Le principe est de constituer une liste de personnages préférés, de leur attribuer un niveau d'appréciation, et de pouvoir ensuite les filtrer et les trier dynamiquement selon leur propriétés, afin d'avoir une vue globale de son état d'appréciation. Les propriétés des personnages sont les suivantes : prénom, nom, sexe, œuvre, acteur, doubleur, image de profil, niveau d'appréciation et commentaire.

     L'utilisateur peut donc créer un compte personnel (Capture 3) et s'y connecter (Capture 4) afin de retrouver sa liste de personnages (Capture 1). Il peut ensuite ajouter (Capture 5 et Capture 6), éditer (Capture 7) ou supprimer (Capture 8) des personnages de sa liste. Enfin, il peut filtrer et trier sa liste, par exemple pour n'afficher que les personnages d'une certaine œuvre triés par niveaux d'appréciation décroissants (Capture 9 à Capture 12).
